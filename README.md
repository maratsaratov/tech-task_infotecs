# ViPNet RAG System

RAG (Retrieval-Augmented Generation) система для поиска и анализа технической документации ViPNet Coordinator HW 5.

## 📋 Описание проекта

Полнофункциональная система поиска по документации ViPNet с использованием гибридного подхода к retrieval-у. Система сочетает BM25 (лексический поиск) и BAAI/bge-m3 (семантический поиск) с использованием Reciprocal Rank Fusion (RRF) для получения оптимальных результатов.

## 🧠 Основные блоки системы

**Блок 1: Установка** - pip-зависимости

**Блок 2: Чанкинг** - TXT → иерархические Parent/Child чанки (3601 parent + 4231 child)

**Блок 3: Индекс** - BM25 + BAAI/bge-m3 → Hybrid Retrieval

**Блок 4: Бенчмарк** - 50 QA-вопросов, 4 метода, метрики Recall/MRR/Context F1

**Блок 5: Анализ** - Визуализация и разбор ошибок

**Блок 6: RAG-чат** - Генерация Qwen2.5-7B-Instruct + интерактивный Q&A

## 📊 Результаты бенчмарка

| Метод | MRR | Recall@1 | Recall@3 | Recall@5 | Context-F1 |
|-------|-----|----------|----------|----------|------------|
| TF-IDF базовый | 0.3637 | 0.28 | 0.42 | 0.50 | 0.1756 |
| BM25 | 0.4687 | 0.36 | 0.54 | 0.68 | 0.1874 |
| Dense (bge-m3) | 0.4207 | 0.30 | 0.56 | 0.62 | 0.1784 |
| **Hybrid RRF** | **0.5487** | **0.44** | **0.66** | **0.72** | **0.2139** |

✅ Гибридный подход показывает лучшие результаты со значением MRR 0.549
